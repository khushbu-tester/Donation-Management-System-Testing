# My Role in Donation Management System (DMS)

As a Junior QA Engineer for this project, I was responsible for ensuring the quality and reliability of the donation workflow across Web and Mobile platforms.

### Key Responsibilities:
* **Requirement Analysis:** Analyzed BRD to identify testable requirements for Zakat, Fitrah, and Harta categories.
* **Test Documentation:** Developed a detailed **Requirement Traceability Matrix (RTM)** and 50+ manual test cases.
* **Functional Testing:** Executed End-to-End (E2E) testing for the **Refund Management Workflow**, ensuring multi-level approvals work correctly.
* **Backend & API Validation:** * Performed **Database Testing** using SQL to verify transaction records.
    * Validated **Payment APIs** via Postman to ensure secure data transfer.
* **Defect Management:** Logged and tracked bugs in JIRA, working closely with developers to ensure timely fixes.
* **Regression Testing:** Conducted regression suites after bug fixes to ensure no new defects were introduced in the payment gateway.

> **Impact:** My testing efforts helped identify 15+ critical bugs before the UAT phase, ensuring a seamless donation experience for the donors.
