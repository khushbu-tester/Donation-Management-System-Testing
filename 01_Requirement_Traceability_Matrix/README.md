# Requirement Traceability Matrix (RTM)

The RTM ensures that all business requirements defined for the Donation Management System are covered by at least one test case. This prevents any functionality from being missed during execution.

| Req ID | Requirement Description | Test Scenario ID | Test Case ID | Status |
|--------|-------------------------|------------------|--------------|--------|
| REQ_01 | Donor should be able to donate via Zakat category | TS_01 | TC_001, TC_002 | Pass |
| REQ_02 | POS terminals should sync transaction data to Web | TS_02 | TC_015 | Pass |
| REQ_03 | Supervisor can Approve/Reject refund requests | TS_03 | TC_022, TC_023 | Pass |
| REQ_04 | Amil can view total received donations in dashboard | TS_04 | TC_030 | Pass |

---

###  Purpose of this RTM:
* **100% Coverage:** Ensures every requirement is tested.
* **Impact Analysis:** If a requirement changes, I know exactly which test cases to update.
* **Audit-Ready:** Professional way to show the testing progress to stakeholders.
